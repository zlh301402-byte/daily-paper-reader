## 摘要
无模型强化学习算法（如近端策略优化，PPO）将环境视为黑箱，从采样奖励中估计策略梯度；这一过程需要数百万次交互，并依赖于高方差的优势估计。当环境动力学可微时，回报是策略参数的端到端可微函数，从而可以通过仿真反向传播精确计算梯度。我们将这种方法称为解析策略梯度（APG），并在四个动力学复杂度递增的连续控制任务上将其与PPO进行比较：一维质点目标到达任务、带避障的二维质点导航任务、二维刚体T形块推任务以及七自由度Franka FR3末端执行器到达任务。两种算法共享相同的模型架构、观测归一化和优化器设置。为了将样本效率与计算效率解耦，我们设计了一个多轴评估协议，记录在环境步数和梯度步数上的性能。我们提出了一种分段反向传播方案，结合MC和基于评论家的引导模式，以缓解长期任务中的梯度退化问题，并展示了关于段长度和引导策略的消融实验。

## Abstract
Model-free reinforcement learning algorithms such as Proximal Policy Optimization (PPO) treat the environment as a black box, estimating policy gradients from sampled rewards; this process demands millions of interactions and relies on high-variance advantage estimates. When environment dynamics are differentiable, the return is an end-to-end differentiable function of the policy parameters, enabling exact gradient computation via backpropagation through simulation. We term this approach Analytic Policy Gradients (APG) and evaluate it against PPO on four continuous control tasks of increasing dynamical complexity: a one-dimensional point-mass target-reaching task, a 2D point-mass navigation task with obstacle avoidance, a 2D rigid-body T-block pushing task, and a 7-DOF Franka FR3 end-effector reaching task. Both algorithms share identical model architectures, observation normalization, and optimizer settings. To decouple sample efficiency from compute efficiency, we design a multi-axis evaluation protocol that records performance against environment steps and gradient steps. We report a segmented backpropagation scheme with MC and critic-based bootstrap modes that mitigates gradient degradation on long-horizon tasks, and present ablations over segment length and bootstrap strategy.

## 2. 方法论：核心思想、关键技术细节、公式/算法
- **核心思想**：当环境转移函数 \(T(s,a)\) 和奖励函数 \(R(s,a)\) 均确定可微时，有限时域期望回报 \(J(\theta) = \sum_{t=0}^{H} \gamma^t R(s_t, a_t)\) 通过重参数化动作 \(a_t = \mu_\theta(s_t) + \sigma_\theta \odot \epsilon_t\) 成为 \(\theta\) 的可微函数。直接对 \(J(\theta)\) 求总导数（式(6)-(9)）得到解析梯度，无需任何蒙特卡洛或价值函数估计。
- **关键技术细节**：
  - **重参数化动作采样**：使用 \(a = \mu_\theta(s) + \sigma_\theta \odot \epsilon\)，使得梯度同时通过均值和对数标准差传播。
  - **分段反向传播**：为缓解长时域梯度退化，将整个 episode 分为 \(K = \lceil H/L_{\text{seg}} \rceil\) 个段，每段内保持计算图，段边界执行 `.detach()` 断开梯度链。段后回报通过 bootstrap 估计。
  - **两种 Bootstrap 模式**：
    - **MC Bootstrap**：从后续段预计算折扣回报（detached），作为当前段的固定引导目标。
    - **Critic Bootstrap**：使用学习到的价值函数 \(V_\phi(s)\) 估计段边界后的未来回报，并保留计算图使得梯度能通过价值函数反向传播。
  - **梯度桥接**：针对 FrankaReach 环境（使用 NVIDIA Warp/Newton 物理引擎），实现自定义 `torch.autograd.Function`，将 Warp 的自动微分通过前向/反向函数桥接到 PyTorch 的 autograd 图。
- **算法流程**（Alg. 1）：每次迭代执行 \(N_{\text{grad}}\) 个梯度步，每个梯度步：重置环境状态，逐段 rollout 并保留计算图，每段结束时 detached 状态并计算 bootstrap 值，累加段损失 \(-\frac{1}{N}\sum (\text{return} + \text{bootstrap})\)，然后反向传播和优化器更新。

## 6. 主要结论与发现
1. **梯度质量优势**：APG 在所有任务上最终回合回报高于 PPO（如 FrankaReach 上 APG 回报0.73 vs PPO -0.09），且到达成功率的提升在简单任务上更明显（PointMassSimple、PushT成功率100%），但复杂任务（PointMassNavigate、FrankaReach）的成功率仍部分，说明密奖励收敛不完全等价于任务成功。
2. **样本效率大幅提升**：在梯度步数维度上，APG 比 PPO 减少 3.9 倍（PushT）到 15.93 倍（FrankaReach）的更新次数；在环境步数上，FrankaReach 减少 4.34 倍，其他任务减少 1.07~1.80 倍（APG 梯度步数更少但环境步数消耗未必更少）。
3. **分段策略至关重要**：MC Bootstrap 在短分段长度（L=10）下仍保持高性能（成功0.9），而 Critic Bootstrap 在 L=10 时完全退化（成功0.0），仅在 L=50 时恢复。MC Bootstrap 是更鲁棒的默认选择，而 Critic Bootstrap 仅在分段足够长时有效。
4. **工程可行性**：通过自定义 `torch.autograd.Function` 实现了外部 GPU 物理引擎（Warp/Newton）与 PyTorch 的梯度桥接，验证了 APG 在非原生可微仿真器中的实用性。

## 7. 优点
- **方法设计亮点**：
  - 统一对比框架，严格隔离变量（仅梯度源不同），确保公平性。
  - 提出多轴评估协议（环境步数 vs 梯度步数），避免算法因更新频率不同导致的比较偏差。
  - 分段反向传播方案清晰，且系统对比两种 bootstrap 模式，提供实用的工程指南。
  - 开源实现和完整环境套件，便于复现和扩展。
- **实验设计亮点**：
  - 环境复杂度递增（1D→2D→2D刚体→7D机械臂），覆盖不同动力学特性。
  - 消融实验针对关键超参数（段长度、bootstrap模式），结论可操作性强。
  - 使用成功率作为更严格的指标补充回报指标，避免密奖励误导。

## 8. 不足与局限
- **环境依赖限制**：APG 必须要求环境动力学可微，无法应用于真实世界训练或非可微仿真（如离散碰撞）。
- **梯度链长度敏感**：长时域仍存在梯度消失/爆炸，分段策略虽缓解但引入额外超参数（段长度）且需调节。
- **计算开销**：维护完整计算图的内存和计算开销比 PPO 的 detached rollout 更大，尤其复杂环境（FrankaReach 需桥接后端）。
- **实验覆盖不足**：
  - 未与基于模型的 RL 方法（DreamerV3、TD-MPC2）或自然梯度方法（TRPO）对比。
  - 消融仅在一个任务（PointMassNavigate）上进行，结论泛化性需进一步验证。
  - 种子数量偏少（主实验推测3个，分段消融仅1个），统计波动可能较大。
- **偏差风险**：所有环境均设计为具有平滑密奖励和确定性/平滑动力学（符合 Suh et al. 理论中路径梯度优势的条件），因此结论可能无法推广到随机或非光滑环境。
- **应用限制**：机器人真实部署需解决不同步问题（仿真与真实动力学差异），论文未涉及 sim-to-real 迁移。
